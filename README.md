# Генератор постов для Threads (Grok / x.ai)

Локальное веб-приложение для ведения экспертного блогерского аккаунта в Threads.
В меню выбирается одна из трёх тем — **Менеджмент**, **Психология**, **Философия** —
и с помощью Grok (API console.x.ai) генерируется готовый к публикации пост.

## Возможности

### Посты (Grok / x.ai)
- Меню выбора темы (3 тематики).
- Необязательное поле «пожелание к посту» (уточнение подтемы, стиля и т.д.).
- Генерация поста через Grok с ограничением под лимит Threads (500 символов).
- Счётчик символов, копирование в буфер, кнопка «Ещё вариант».
- Окно редактирования поста — вручную и с помощью Grok (правки идут по цепочке).

### Изображения (WaveSpeed / Nano Banana Pro)
- **Текст → изображение**: генерация по текстовому описанию.
- **Редактирование изображения** (image-to-image): загрузка исходного изображения + описание изменений.
- Выбор параметров перед генерацией: соотношение сторон (1:1 … 21:9), разрешение (1k/2k/4k), формат (png/jpeg).
- Скачивание результата и кнопка «Редактировать это» (результат становится исходником для правки).

## Запуск

1. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Создайте файл `.env` на основе `.env.example` и вставьте свои ключи:

   ```bash
   cp .env.example .env
   # XAI_API_KEY — ключ с https://console.x.ai (посты)
   # WAVESPEED_API_KEY — ключ с https://wavespeed.ai (изображения)
   ```

3. Запустите сервер:

   ```bash
   python app.py
   ```

4. Откройте в браузере: **http://127.0.0.1:5000**

## Публичная ссылка (деплой)

> ⚠️ GitHub Pages НЕ подходит: это статический хостинг, он не запускает Python/Flask
> и не может безопасно хранить ключ Grok. По ссылке Pages показывает только README.

Приложение нужно разместить на хостинге, который запускает Python. Пример — [Render](https://render.com) (бесплатный план):

1. На render.com → **New** → **Web Service** → подключите репозиторий `shaktiijee/Aproject`.
2. Render сам подхватит `render.yaml` (build: `pip install -r requirements.txt`, start: `gunicorn`).
3. В разделе **Environment** задайте переменные `XAI_API_KEY` (x.ai) и `WAVESPEED_API_KEY` (WaveSpeed) — ключи НЕ хранятся в коде.
4. Нажмите **Deploy** — получите публичную ссылку вида `https://threads-post-generator.onrender.com`.

## Настройки (`.env`)

| Переменная    | Назначение                                   | По умолчанию |
| ------------- | -------------------------------------------- | ------------ |
| `XAI_API_KEY` | Ключ API с console.x.ai (обязательно для постов) | —        |
| `WAVESPEED_API_KEY` | Ключ WaveSpeed (обязательно для изображений) | —      |
| `XAI_MODEL`   | Модель Grok                                   | `grok-4.5`   |
| `HOST`        | Адрес локального сервера                      | `127.0.0.1`  |
| `PORT`        | Порт локального сервера                       | `5000`       |
